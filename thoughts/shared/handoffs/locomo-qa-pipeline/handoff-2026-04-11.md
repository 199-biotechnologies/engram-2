## What Was Accomplished This Session

### 1. Confound isolation — the MLX reranker regression was TWO bugs

Ran 6 controlled experiments on a deterministic 50-question LoCoMo slice:

| Exp | Reranker | Embed | Judge | Accuracy | Latency |
|---|---|---|---|---:|---:|
| 1 | Cohere v3.5 | embed-001 | gpt-5.4 (lenient) | **84.0%** | 4.6s/q |
| 2 | Cohere v3.5 | embed-2-preview | gpt-5.4 (lenient) | 78.0% | 8.5s/q |
| 3 | zerank-2 MLX | embed-001 | gpt-5.4 (lenient) | 45.8% (24q) | 21.4s/q |
| 5 | No rerank | embed-001 | gpt-5.4 (lenient) | 72.0% | 3.2s/q |
| 6 | **jina-v3 MLX** | embed-001 | gpt-5.4 (lenient) | **84.0%** | 8.0s/q |
| 7 | Cohere v3.5 | embed-001 | claude-sonnet-4.6 | **66.0%** | 7.6s/q |
| STRICT | Cohere v3.5 | embed-001 | gpt-5.4 (strict) | **32.0%** | 5.0s/q |

**Root causes found:**
- **zerank-2 is the wrong model for LoCoMo** (-38 pts vs Cohere, worse than no rerank). Biomed-optimized, bad at conversational QA.
- **The judge was inflating scores by 18-52 points.** GPT-5.4 accepted partial answers (1 of 3 list items), vague paraphrases, hedging. Claude-sonnet-4.6 was stricter. New strict judge prompt → honest 32%.
- **embed-2-preview showed -6 pts** vs embed-001, but Boris says this is a bench issue not a model issue — embed-2 is superior. Needs further investigation.

### 2. Replaced zerank-2 with jina-reranker-v3-mlx

- Built `crates/engram-rerank/python/jina_reranker_v3_server.py` — 306-line MLX sidecar
- jina-v3 is 0.6B (7× smaller than zerank-2's 4B), official MLX weights from Jina
- Uses listwise architecture: all docs in one prompt with `<|embed_token|>` / `<|rerank_token|>` markers, MLP projector, cosine similarity scoring
- **Matches Cohere exactly** at 84.0% (42/50) on LoCoMo with the lenient judge
- Same HTTP contract as zerank-2 (POST /rerank, GET /health), drop-in replacement
- Committed: `b7eb0b4`

## Key Decisions Made

1. **zerank-2 is dead for LoCoMo.** Don't revisit. Use jina-v3-mlx as the local reranker.
2. **jina-reranker-v3-mlx is the shipped local reranker.** 0.6B, official MLX weights, matches Cohere. But it has NO Rust client yet — only Python sidecar + zerank_local.rs Rust client pointing at the same HTTP endpoint.
3. **The strict judge is the honest judge.** 32% is the real number. The 84% with lenient judge masks retrieval failures (partial answers accepted).
4. **embed-2-preview regression needs investigation.** Boris believes embed-2 is superior. If our bench shows it worse, the bench is wrong. Could be: (a) the config-file Gemini key might use a different project with different model access, (b) embed-2-preview might need different `taskType` or `outputDimensionality`, (c) the embedding cache from prior runs might be mixing models.
5. **MemPalace's speaker-owned closets** (one chunk per speaker per session) is a genuinely good idea that solved their adversarial category. Worth considering for engram's chunking strategy.
6. **The Gemini API key in env (`GEMINI_API_KEY`) is expired.** The config-file key (ending `lZ1l9M`) works. All bench runs this session used `unset GEMINI_API_KEY` to fall back to the config key.
